Commons installs two ways from the same source. Use the **React / HTML**
switcher (beside the theme control) to follow the path that matches your stack:

- **React** — the component library plus the CLI that copies source into your repository.
- **HTML** — `commons.css` and a small runtime for any server-rendered stack (Drupal, WordPress, .NET, Rails, plain HTML) — no React and no build step.

## Requirements

<Framework only="react">

- **Node.js 22 or later** for the CLI and build tooling.
- **React 19** for `@21stgov/commons-react` components.
- **Tailwind CSS v4** — Commons components are styled with utilities that the token bridge maps onto Commons design tokens.

</Framework>

<Framework only="html">

- **No build step and no React.** You serve a few stylesheets and one script.
- **Node / npm are optional** — use them to pull the packages from npm, or vendor the built CSS and JS files into your project and serve them from your own host.

</Framework>

## Install

<Framework only="react">

`init` inspects your project, writes a `commons.json` configuration, and sets
up the token and CSS entry points:

<Tabs groupId="package-manager" persist items={["npm", "pnpm", "yarn", "bun"]}>

<Tab value="npm">

```sh
npx @21stgov/commons init
```

</Tab>

<Tab value="pnpm">

```sh
pnpm dlx @21stgov/commons init
```

</Tab>

<Tab value="yarn">

```sh
yarn dlx @21stgov/commons init
```

</Tab>

<Tab value="bun">

```sh
bunx @21stgov/commons init
```

</Tab>

</Tabs>

Add `--yes` for non-interactive setup (CI, scripts, agents), and `--json` for
a versioned machine-readable result envelope.

</Framework>

<Framework only="html">

**Fastest — the CDN.** One stylesheet (design tokens, themes, and the `.cui-*`
component classes) and the runtime, which auto-enhances any `.cui-*` markup it
finds on the page:

<CdnLinks />

Versioned paths are immutable — pin the exact version you tested against.

**Self-host or pin via npm.** Install the token, core, component-CSS, and
runtime packages (or vendor the built files and skip this step):

<Tabs groupId="package-manager" persist items={["npm", "pnpm", "yarn", "bun"]}>

<Tab value="npm">

```sh
npm install @21stgov/commons-tokens @21stgov/commons-core @21stgov/commons-css @21stgov/commons-js
```

</Tab>

<Tab value="pnpm">

```sh
pnpm add @21stgov/commons-tokens @21stgov/commons-core @21stgov/commons-css @21stgov/commons-js
```

</Tab>

<Tab value="yarn">

```sh
yarn add @21stgov/commons-tokens @21stgov/commons-core @21stgov/commons-css @21stgov/commons-js
```

</Tab>

<Tab value="bun">

```sh
bun add @21stgov/commons-tokens @21stgov/commons-core @21stgov/commons-css @21stgov/commons-js
```

</Tab>

</Tabs>

Load the stylesheets in order, plus the runtime. The `commons.global.js` build
auto-enhances any `.cui-*` markup on the page — no code to write:

```html
<link rel="stylesheet" href="/assets/commons-tokens.css" />
<link rel="stylesheet" href="/assets/commons-core.css" />
<link rel="stylesheet" href="/assets/commons.css" />
<script src="/assets/commons.global.js" defer></script>
```

With a bundler, import the runtime and call `enhance()` yourself instead of the
global script:

```js
import { enhance } from '@21stgov/commons-js'
enhance()
```

</Framework>

## Add a component

<Framework only="react">

```sh
npx @21stgov/commons add button
```

The CLI copies the component source into your repository — you own the code —
and installs any npm dependencies it needs. Registry dependencies (like the
`cn` class-name utility) are resolved automatically. Preview everything first
with `--dry-run`.

Browse the [component catalog](/docs/components) for what's available.

</Framework>

<Framework only="html">

There is no per-component install step — every class ships in `commons.css`.
Open any [component](/docs/components), switch its example to **HTML**, and copy
the markup from the **Code** tab into your template. Interactive components
(dialogs, menus, tabs, accordions, and the rest) are wired up automatically once
`commons-js` is loaded.

Browse the [component catalog](/docs/components) for what's available.

</Framework>

## Wire up the CSS

<Framework only="react">

Commons layers its stylesheets deliberately. If you wire the CSS yourself,
the order matters:

```css
@import '@21stgov/commons-fonts/index.css'; /* optional: system-font fallback works */
@import '@21stgov/commons-tokens/index.css'; /* --cui-* custom properties + themes */
@import '@21stgov/commons-core/index.css'; /* accessible reset (unlayered on purpose) */
@import 'tailwindcss'; /* the engine */
@import '@21stgov/commons-tokens/tailwind.css'; /* the @theme bridge — MUST come last */
```

The bridge must come after `tailwindcss`: it removes Tailwind's stock color
palette (`--color-*: initial`) so nothing can silently bypass the Commons
contrast contract, and that removal only works when it follows Tailwind's own
theme declarations. Semantic utilities reference `var(--cui-*)` at runtime, so
switching themes via `data-theme` keeps working without a rebuild.

</Framework>

<Framework only="html">

No Tailwind and no bridge — import the plain stylesheets in this order (tokens
first; the core and the component classes consume the `--cui-*` variables):

```css
@import '@21stgov/commons-fonts/index.css'; /* optional: system-font fallback works */
@import '@21stgov/commons-tokens/index.css'; /* --cui-* custom properties + themes */
@import '@21stgov/commons-core/index.css'; /* accessible reset */
@import '@21stgov/commons-css'; /* the .cui-* component classes */
```

Every rule degrades gracefully to system colors and UA defaults if the tokens
are missing, so a partial adoption never breaks the page.

</Framework>

## Fonts

`@21stgov/commons-fonts` ships self-hosted Atkinson Hyperlegible Next and
Atkinson Hyperlegible Mono variable fonts — no third-party font CDN. On the
framework-agnostic path, the CDN serves the same fonts as a single self-contained
stylesheet (the `fonts.css` `<link>` shown above); on the React path, import
`@21stgov/commons-fonts/index.css`. The Atkinson Hyperlegible font files are
licensed separately under the SIL Open Font License, Version 1.1. Skipping the
fonts is safe either way: every token font stack declares system-font fallbacks.
