A large share of local-government sites are server-rendered — PHP, Drupal,
WordPress, .NET (Razor/MVC), Java (Spring/JSP), Rails, or plain HTML. Commons is
built for that reality, and the non-React path is a first-class use case, not an
afterthought. You can adopt it at three depths — from a light restyle to the
full component set with interactivity — all with no React dependency and no
build step.

## What works everywhere

| Package | Framework-agnostic? | What you get |
| --- | --- | --- |
| `@21stgov/commons-tokens` | ✅ Yes | `--cui-*` CSS custom properties: color, type, spacing, focus, motion, and the light / dark / high-contrast themes. |
| `@21stgov/commons-core` | ✅ Yes | A modern accessible reset, base element styles, a skip link, `.cui-sr-only`, global `:focus-visible` rings, forced-colors hardening, and 44px `.cui-target` sizing. |
| `@21stgov/commons-css` | ✅ Yes | `commons.css` — the `.cui-*` component classes (button, alert, card, table, dialog, menu…), generated 1:1 from the React components so the look matches. |
| `@21stgov/commons-js` | ✅ Yes | A small progressive-enhancement runtime. `enhance()` wires the interactive behaviors onto `.cui-*` markup; a `<script>` build auto-enhances the page. |
| `@21stgov/commons-fonts` | ✅ Yes | Self-hosted Atkinson Hyperlegible Next + Mono variable fonts (no font CDN). |
| `@21stgov/commons-react` | ⚠️ React only | The React component library. Not needed for the paths on this page. |

## Three ways in

1. **Token-only restyle** — import the tokens and core, then style your own existing markup with the `--cui-*` variables. Lightest touch; works today, everywhere.
2. **The `.cui-*` component classes** — add `commons.css` and write Commons' component markup directly (`<button class="cui-button">…`).
3. **Add interactivity** — load `commons-js`, and the interactive components (dialogs, menus, tabs, accordions, carousels, comboboxes, data tables…) come alive with no custom JavaScript.

The [Installation](/docs/installation) page (switch it to **HTML**) has the exact
install commands and load order. The rest of this page covers what each depth
gives you. Everything degrades gracefully — a partial adoption never breaks the
page.

## Token-only: restyle your markup

Style your existing server-rendered markup with the `--cui-*` variables. Your
PHP/Drupal/WordPress/.NET/Java templates keep their markup and pick up Commons'
contrast-checked colors, spacing, and type.

```css
.btn-primary {
  background: var(--cui-action-primary-bg);
  color: var(--cui-action-primary-text);
  min-block-size: 2.75rem; /* 44px target */
  padding-inline: var(--cui-spacing-2);
  border-radius: var(--cui-radius-sm);
}
```

Theme switching is a single attribute on the root — no JavaScript required (it
follows the OS via `prefers-color-scheme` / `prefers-contrast` by default):

```html
<html data-theme="dark">   <!-- or "light", or "high-contrast"; omit to follow the OS -->
```

See [Theming](/docs/theming) for the full token architecture and how to rebrand
for your agency.

## The `.cui-*` component classes

`commons.css` ships the component classes the React components compile to, so you
can write the same components as plain HTML:

```html
<button type="button" class="cui-button cui-button--primary">Submit application</button>

<div class="cui-alert cui-alert--warning" role="alert">
  <!-- … -->
</div>
```

The classes are generated from the React components' own style configs, so the
vanilla output stays 1:1 with React instead of drifting. The quickest way to get
correct markup is to open a [component](/docs/components), switch its example to
**HTML**, and copy from the **Code** tab.

## Interactivity with commons-js

Static components (button, badge, card, alert…) need nothing beyond the CSS.
Interactive ones (dialog, dropdown, menu, tabs, accordion, carousel, combo box,
data table, and ~40 more) need keyboard behavior, focus management, and ARIA
state wired up — that is what `@21stgov/commons-js` does.

- **Drop-in:** load the `commons.global.js` build and it enhances every `.cui-*`
  region on the page automatically on load. For markup you inject later, call
  `window.Commons.enhance()` again.
- **Bundler:** `import { enhance } from '@21stgov/commons-js'` and call
  `enhance()` (optionally scoped to a root element).

The behaviors follow the same normative accessibility contract published on each
[component's page](/docs/components) and in the registry JSON, so the keyboard
and screen-reader experience matches the React components.

## Accessibility utilities

The core ships the always-on guarantees as plain classes you can drop into any
template:

- `.cui-skip-link` — a "skip to main content" link (first focusable element in `<body>`)
- `.cui-sr-only` — visually-hidden, screen-reader-only text
- `.cui-target` — enforces the 44×44px minimum touch target
- Global visible `:focus-visible` rings, forced-colors support, and reduced-motion handling apply automatically.

## Adoption modes, at a glance

- **Tokens + core only** — restyle an existing server-rendered site with the `--cui-*` variables. Works today, everywhere.
- **+ commons.css** — use the `.cui-*` component classes for static components (buttons, cards, alerts, tables…). Works today.
- **+ commons-js** — add the interactive components (dialogs, menus, tabs…) with no React. Works today.
- **Full React** — a React/Vite/Next/Astro app using the component library and the CLI. See [Installation](/docs/installation).

You can mix these freely and start anywhere. If you are adopting Commons in one
of these stacks and hit a gap, please [open an
issue](https://github.com/21stgov/commons) — the non-React path is a first-class
use case, not an afterthought.
